# Code Review

커밋되지 않은 변경사항에 대한 종합 보안+품질 리뷰:

1. 변경 파일 확인: `git diff --name-only HEAD`

2. 각 변경 파일에 대해 검사:

**보안 이슈 (CRITICAL):**
- 하드코딩된 자격증명, API 키, 토큰
- SQL 인젝션 취약점
- XSS 취약점
- 입력 검증 누락
- 안전하지 않은 의존성
- 경로 순회 위험

**코드 품질 (HIGH):**
- 50줄 초과 함수
- 800줄 초과 파일
- 4단계 초과 중첩
- 에러 핸들링 누락
- console.log / print 문
- TODO/FIXME 주석
- 공개 API에 docstring 누락

**베스트 프랙티스 (MEDIUM):**
- 변이(mutation) 패턴 (불변 방식으로)
- 새 코드에 테스트 누락
- 접근성 이슈 (a11y)

3. 리포트 생성:
   - 심각도: CRITICAL, HIGH, MEDIUM, LOW
   - 파일 위치 및 라인 번호
   - 이슈 설명
   - 수정 제안

4. CRITICAL 또는 HIGH 이슈 발견 시 커밋 차단 권고

보안 취약점이 있는 코드는 절대 승인하지 마라!
