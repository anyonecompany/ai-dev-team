# /explore - 코드베이스 탐색

## 파라미터

| 인자 | 설명 | 기본값 |
|------|------|--------|
| 검색어 | 탐색할 키워드/함수명/패턴 (필수) | - |
| `--auto` | 자동 3회 정제 | 꺼짐 |
| `--deps` | 의존성 추적 포함 | 꺼짐 |
| `--scope 경로` | 탐색 범위 제한 | 프로젝트 루트 |

## 실행 흐름

### 1단계: 키워드 확장
auth → login, session, jwt, token, authenticate 등

### 2단계: 초기 탐색
1. **파일명 검색** — Glob으로 `*{검색어}*` 패턴
2. **코드 내용 검색** — Grep으로 정의/사용처
3. **Git 히스토리** — `git log --grep="{검색어}"`

### 3단계: 필터링
- 상위 15개 파일 선정
- 각 파일 구조 분석 (함수, 클래스, export)

### 4단계: --deps 의존성 추적
- **상향**: 이 코드를 사용하는 곳
- **하향**: 이 코드가 의존하는 곳

### 5단계: 결과 출력
```
탐색 결과: "{검색어}"
  파일: N개
  정의: M개
  사용처: K개

핵심 파일:
  1. [경로] - [역할]
  2. [경로] - [역할]

의존성: (--deps 시)
  상향: N개 파일이 사용
  하향: M개 모듈에 의존
```
