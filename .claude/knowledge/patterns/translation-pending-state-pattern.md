# 번역 미완료 상태 처리 패턴

> 등록일: 2026-03-24
> 프로젝트: Portfiq
> 적용 범위: 번역/가공이 필요한 모든 텍스트 필드

## 패턴

번역 중인 필드는 **빈값("") 유지** + UI에서 **`isNotEmpty` 조건으로 위젯 자체를 숨기는 방식**이 정답.

### 백엔드
```python
# 번역 미완료 시 — 빈값 유지
if not summary_3line:
    summary_3line = ""  # 영문 원제를 넣지 않음
```

### Flutter UI
```dart
// summary가 있을 때만 위젯 표시
if (item.summary3line.isNotEmpty) ...[
  const SizedBox(height: 8),
  Text(item.summary3line.replaceAll('\n', ' '), ...),
],
// 없으면 → 위젯 자체가 없음 (헤드라인 + 배지만 보임)
```

## 대안 비교

| 방식 | 장점 | 단점 |
|------|------|------|
| **빈값 + 위젯 숨김** (채택) | 깔끔, 추가 UI 불필요 | 요약 존재 여부 알 수 없음 |
| placeholder "번역 중..." | 사용자에게 상태 전달 | 화면에 불필요한 텍스트 증가 |
| 영문 원제 fallback | 빈 상태 방지 | 한국어 앱에서 영문 노출 = 버그 |
| 로딩 스피너 | 명시적 상태 | 과잉 — 뉴스 카드 하나하나에 스피너는 과함 |

## 적용 원칙

1. 번역/가공 대상 필드의 fallback에 원문(다른 언어)을 사용하지 않는다
2. UI에서 빈값 체크 후 위젯 숨기기로 처리한다
3. placeholder 텍스트("번역 중...")보다 숨기는 게 UX상 낫다
4. 번역 완료 후 캐시/DB에 결과를 반드시 반영한다 (캐시 무효화 포함)
