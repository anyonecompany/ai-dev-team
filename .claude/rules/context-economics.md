# Context Economics Rules

## 컨텍스트 예산 원칙
- 200k 컨텍스트 창은 MCP/도구 수에 따라 실질적으로 70k까지 축소됨
- 항상 컨텍스트 잔여량을 의식하고 작업 순서를 계획할 것
- 컨텍스트 50% 소진 시 `/session-save` 실행 후 계속 진행
- 컨텍스트 80% 소진 시 반드시 컴팩션 후 새 세션에서 재개

## MCP 규율 (필수)
- 설정에 MCP를 추가할 때: 비활성화 상태로 등록
- **활성 MCP: 10개 이하** / **활성 도구: 80개 이하** 유지
- 현재 태스크와 무관한 MCP는 즉시 비활성화
- `/mcp-status` 커맨드로 세션 시작 전 항상 확인

## 서브에이전트 위임 원칙
- 서브에이전트에게 태스크를 위임할 때 반드시 **목적/이유(PURPOSE)** 를 함께 전달
- 잘못된 위임: "src/api.py 파일의 validate 함수 수정해줘"
- 올바른 위임: "사용자 인증 흐름 개선이 목적임. src/api.py의 validate 함수에서 JWT 만료 처리 누락 버그 수정해줘"
- 서브에이전트는 summary만 반환 — 전체 결과물 덤프 금지

## 탐색 비용 최소화
- 새 세션 시작 시 `.claude/knowledge/codemap.md` 먼저 로드
- 코드베이스 탐색에 컨텍스트 소진 금지 — codemap 활용
- 코드맵 없을 경우 `/codemap-update` 선행 실행
