# 에이전트 운영 규칙

> 출처: claude-forge agents-v2.md + ai-dev-team 기존 규칙 통합

## 에이전트 즉시 사용 (프롬프트 불필요)

다음 상황에서는 사용자 요청 없이 즉시 적절한 방식을 적용:
1. 복잡한 기능 요청 → **설계 먼저** (`/plan`)
2. 코드 작성/수정 직후 → **코드 리뷰** (`/code-review`)
3. 버그 수정 또는 새 기능 → **TDD 접근**
4. 아키텍처 결정 → **설계 문서화**

## 병렬 실행

독립적 작업은 **반드시** 병렬로:

```
좋은 예: 3개 에이전트 동시 실행
  1. 보안 분석: auth.ts
  2. 성능 리뷰: cache 시스템
  3. 타입 체크: utils.ts

나쁜 예: 불필요한 순차 실행
  에이전트 1 완료 → 에이전트 2 → 에이전트 3
```

## 서브에이전트 vs Agent Teams

| | 서브에이전트 | Agent Teams |
|---|---|---|
| 컨텍스트 | 자체 윈도우, 결과만 반환 | 자체 윈도우, 완전 독립 |
| 소통 | 메인 에이전트에만 보고 | 리더 허브+스포크 |
| 조율 | 메인이 관리 | 리더가 조율 + 공유 태스크 |
| 적합 | 결과만 필요한 집중 작업 | 토론/협업 필요한 복잡 작업 |
| 토큰 비용 | 낮음 | 높음 (팀원 수에 비례) |

## Notion 보고 필수

모든 에이전트 작업은 Notion에 보고:
- 태스크 시작/완료: `report_task_done()`
- 기술 결정: `report_decision()`
- 후속 태스크: `add_task()`
- 통합 보고: `report_completion()`

**project_name은 반드시 지정. 예외 없음.**
