# 인공지능기본법 준수 가이드라인

> 버전: 1.0.0
> 최종 갱신: 2026-02-06
> 근거법: 인공지능 발전과 신뢰 기반 조성 등에 관한 기본법 (2026.01.22 시행)
> 참고: https://www.sw.or.kr/AI_act_helpdesk/main.jsp

---

## 개요

본 문서는 AI Dev Team 플랫폼에서 개발하는 모든 AI 기반 서비스가 **한국 인공지능기본법**을 준수하도록 보장하기 위한 필수 가이드라인입니다. 모든 에이전트는 코드 생성, 설계, 기획 시 본 가이드라인을 반드시 따라야 합니다.

---

## 1. 적용 범위 판단 (모든 프로젝트 시작 시 필수)

### 1-1. 고영향 AI 해당 여부 확인

다음 11개 분야에서 AI가 의사결정에 관여하는 경우 **고영향 AI**로 분류됩니다:

| 번호 | 분야 | 예시 |
|------|------|------|
| 1 | 에너지 | 전력 수요 예측, 스마트 그리드 |
| 2 | 의료/보건 | AI 진단, 의료 영상 분석 |
| 3 | 금융 | 신용평가, 대출 심사, 보험 심사 |
| 4 | 교통 | 자율주행, 교통 관제 |
| 5 | 고용/채용 | AI 이력서 스크리닝, 면접 평가 |
| 6 | 교육 | AI 학습 평가, 맞춤 교육 |
| 7 | 공공안전 | 범죄 예측, 감시 시스템 |
| 8 | 사법 | 법률 판단 보조, 양형 예측 |
| 9 | 복지 | 수급자 선정, 복지 서비스 배분 |
| 10 | 환경 | 환경 영향 예측, 오염 모니터링 |
| 11 | 통신 | 네트워크 관리, 콘텐츠 필터링 |

**판단 기준:**
- AI 오작동 시 **생명/신체 안전** 또는 **기본권**에 중대한 영향을 미치는가?
- **불공정한 차별** 가능성이 있는가?
- 최종 의사결정에 **인간의 개입 없이** AI가 직접 결정하는가?

> **참고**: 인간이 최종 의사결정을 통제하는 경우 고영향 AI에서 제외될 수 있음

### 1-2. 생성형 AI 해당 여부 확인

다음 중 하나라도 해당되면 **생성형 AI**로 분류됩니다:
- 텍스트, 이미지, 영상, 음향 등 새로운 결과물을 생성
- LLM(대형언어모델) 기반 서비스 제공
- 딥페이크 등 실제와 구분하기 어려운 콘텐츠 생성

## 2. 투명성 확보 의무 (모든 AI 서비스 필수)

### 2-1. AI 사용 고지 (필수)

**모든 AI 기반 서비스**는 이용자에게 AI가 사용되고 있음을 **사전에 고지**해야 합니다.

**구현 요구사항:**
```
[필수] 서비스 내 "AI 기반 운영" 고지 UI 요소 포함
[필수] 고지 방법 중 최소 1가지 이상 적용:
  - 제품/서비스 내 직접 기재
  - 이용약관에 명시
  - 화면 내 표시 (배너, 라벨, 아이콘 등)
  - 서비스 제공 장소에 게시
[필수] 고지 내용: AI 기술 활용 사실, AI 역할 범위
```

**면제 조건:**
- 제품명/화면으로 AI 사용이 명백한 경우 (예: "AI 번역기")
- 사업자 내부 업무용으로만 사용되는 경우

### 2-2. 생성형 AI 투명성 (생성형 AI 해당 시 필수)

```
[필수] AI 생성 결과물에 워터마크 표시
  - 인간이 인지 가능한 형태 (예: "AI로 생성됨" 라벨)
  - 기계가 판독 가능한 형태 (메타데이터, 비가시 워터마크)
[필수] 딥페이크 등 실제와 구분 어려운 콘텐츠 명확 고지
[필수] 사용자가 AI 생성 여부를 확인할 수 있는 수단 제공
```

**위반 시 과태료: 3,000만원 이하**

## 3. 안전성 확보 의무

### 3-1. 기본 안전성 (모든 AI 서비스)

```
[필수] AI 시스템의 예상 가능한 위험 식별 및 문서화
[필수] 위험 완화 조치 구현 및 기록
[필수] 사고 발생 시 대응 체계 구축
[필수] 정기적 안전성 점검 및 업데이트
```

### 3-2. 최첨단 AI 안전성 (학습 연산량 10^26 FLOP 이상)

```
[필수] 개발~폐기 전 생애주기 위험 식별/평가/완화 체계
[필수] AI 안전사고 모니터링 및 대응 체계
[필수] 이행 결과를 과학기술정보통신부장관에게 정기 제출
```

## 4. 고영향 AI 사업자 특별 책무

### 4-1. 위험관리 방안 (고영향 AI 해당 시 필수)

```
[필수] 위험관리 정책 수립 및 공개
[필수] 위험관리 조직 체계 구성
[필수] 홈페이지 등에 위험관리 정책 공개
[필수] 정기적 위험 재평가 실시
```

### 4-2. 이용자 보호 방안 (고영향 AI 해당 시 필수)

```
[필수] AI 의사결정 근거 설명 기능 (설명 가능성, XAI)
[필수] 이용자가 AI 판단 결과에 이의 제기 가능한 메커니즘
[필수] 투명성 강화 조치 및 홈페이지 공개
[필수] 개인정보 처리 관련 동의 절차
```

## 5. AI 영향평가

### 5-1. 평가 항목 (권장, 고영향 AI는 사실상 필수)

```
[필수] 영향 대상 식별 (특정 집단, 취약 계층 포함)
[필수] 영향받을 기본권 유형 식별 (평등권, 프라이버시권, 알 권리 등)
[필수] 사회적/경제적 영향 범위 평가
[필수] AI 시스템 사용 패턴 분석
[필수] 평가 지표 및 기준 정의
[필수] 위험 예방 및 손실 회복 방안 수립
[필수] 개선 계획 수립
[필수] 서비스 출시 전 평가 완료
```

## 7. 에이전트별 준수 사항

### PM-Planner
- PRD 작성 시 **고영향 AI 해당 여부** 판단 섹션 포함
- 요구사항에 **투명성/안전성 요건** 명시
- **영향평가 필요 여부** 판단 및 평가 항목 사전 정의

### Architect
- 시스템 설계 시 **AI 고지 메커니즘** 아키텍처 포함
- **워터마크/라벨링 시스템** 설계
- **설명 가능성(XAI)** 인터페이스 설계
- **로깅/감사 추적** 시스템 아키텍처 포함

### Designer
- UI에 **AI 사용 고지** 요소 반드시 포함
- **AI 생성 콘텐츠 표시** UI 패턴 적용
- **이의 제기 메커니즘** UI 설계
- 접근성 고려한 고지 디자인

### BE-Developer
- **AI 의사결정 로깅** API 구현
- **워터마크/메타데이터** 삽입 로직 구현
- **편향성 테스트** 엔드포인트 구현
- **영향평가 데이터** 수집 API 구현
- AI 응답에 **출처/근거 정보** 포함

### FE-Developer
- **AI 사용 고지 배너/라벨** 컴포넌트 구현
- **AI 생성 콘텐츠 표시** UI 구현
- **사용자 이의 제기** 인터페이스 구현
- **투명성 정보 표시** 페이지 구현

### AI-Engineer
- **모델 투명성** 문서화 (모델 카드)
- **편향성 테스트** 파이프라인 구축
- **설명 가능성(XAI)** 기능 구현
- **워터마킹** 기술 적용
- 학습 데이터 **출처/라이선스** 관리

### QA-DevOps
- **투명성 요건 테스트** (고지 UI 존재 확인)
- **워터마크 검증** 테스트
- **편향성 테스트** 자동화
- **컴플라이언스 체크** CI/CD 파이프라인 통합
- **영향평가 보고서** 자동 생성

### Security-Developer
- **개인정보 처리** 적법성 검증
- **데이터 암호화** 및 접근 통제
- **감사 로그** 무결성 보장
- **AI 모델 보안** (적대적 공격 방어)
